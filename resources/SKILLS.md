

# Skills

## Plan Webservice Tests

1. Read the provided webservice definition URL.
2. Extract endpoints, methods, parameters, request bodies, responses, schemas, and authentication.
3. Create structured test cases with:
   - id
   - goal
   - method
   - endpoint
   - input data
   - expected result
   - priority
4. Include positive, negative, boundary, validation, and authentication tests.
5. Avoid destructive tests unless explicitly approved.
6. Do not invent missing details. State assumptions.
7. Use tool `save_test_cases` with the complete test case list.
8. Do not return a final answer until the save tool succeeds.
